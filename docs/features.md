# 待办事项应用 - 功能规格文档

## 1. 实体定义

### 用户
使用待办事项应用的个人

### 待办事项
- id: 唯一标识符
- text: 事项内容
- done: 完成状态

### 统计数据
- totalCompleted: 历史累计完成数
- todayCompleted: 今日完成数
- todayDate: 当前日期
- currentStreak: 连续完成天数
- lastCompletedDate: 上次完成日期
- achievements: 已解锁成就列表

### 成就
- id: 唯一标识符
- name: 成就名称
- desc: 成就描述
- icon: 成就图标
- require: 解锁条件

---

## 2. 用户故事

### 用户故事 1: 添加待办事项

**作为** 用户  
**我想要** 在应用中快速添加新的待办事项  
**以便** 记录我需要完成的任务

验收标准:
- 可以在输入框中输入待办事项内容
- 按下添加按钮或回车键可以添加事项
- 添加成功后输入框自动清空
- 新事项显示在列表顶部

---

### 用户故事 2: 查看待办事项列表

**作为** 用户  
**我想要** 查看我所有的待办事项  
**以便** 了解还有哪些任务需要完成

验收标准:
- 显示待完成事项的数量
- 显示已完成事项的数量
- 待办事项显示在最上方
- 空状态显示友好提示

---

### 用户故事 3: 标记事项完成

**作为** 用户  
**我想要** 点击复选框标记事项为已完成  
**以便** 记录我已经完成的任务

验收标准:
- 点击复选框可以切换完成状态
- 已完成事项显示删除线效果
- 完成时显示鼓励语
- 今日完成数自动加1

---

### 用户故事 4: 删除待办事项

**作为** 用户  
**我想要** 删除不再需要的待办事项  
**以便** 保持列表的简洁

验收标准:
- 悬停时显示删除按钮
- 点击删除按钮移除事项
- 删除已完成事项时总完成数减1

---

### 用户故事 5: 清除已完成的待办

**作为** 用户  
**我想要** 一键清除所有已完成的事项  
**以便** 快速清理已完成任务

验收标准:
- 显示清除已完成按钮
- 点击后移除所有已完成事项
- 历史累计完成数相应减少

---

### 用户故事 6: 查看进度

**作为** 用户  
**我想要** 查看当前待办事项的完成进度  
**以便** 了解任务完成情况

验收标准:
- 显示进度条
- 显示百分比或完成状态文字
- 无事项时显示提示文字

---

### 用户故事 7: 跟踪每日完成数

**作为** 用户  
**我想要** 查看今日完成了多少任务  
**以便** 了解每天的效率

验收标准:
- 在首页显示今日完成数量
- 每天自动重置计数
- 跨日期后从0开始计数

---

### 用户故事 8: 跟踪连续完成天数

**作为** 用户  
**我想要** 查看连续完成任务的天数  
**以便** 保持良好的习惯

验收标准:
- 在首页显示连续天数
- 连续每天完成至少一个任务则加1
- 中断一天则重置为1

---

### 用户故事 9: 查看成就

**作为** 用户  
**我想要** 查看我的成就和徽章  
**以便** 获得完成任务的动力

验收标准:
- 点击成就按钮打开模态框
- 显示已解锁/未解锁成就数量
- 未解锁成就显示为锁定状态

---

### 用户故事 10: 解锁成就获得奖励

**作为** 用户  
**我想要** 在完成任务时看到成就解锁动画  
**以便** 获得成就感和鼓励

验收标准:
- 完成第一个任务时弹出鼓励语
- 达成里程碑时显示庆祝动画
- 解锁成就时显示成就卡片动画

---

## 4. 交互方案

### 4.1 添加待办
输入内容 -> 点击添加按钮或按回车 -> 添加到列表顶部 -> 清空输入框 -> 更新统计

### 4.2 完成待办
点击复选框 -> 切换完成状态 -> 如果从未完成变为完成则执行以下流程 -> todayCompleted加1 -> 检查成就解锁条件 -> 显示鼓励语弹窗 -> 检查里程碑 -> 显示里程碑庆祝动画 -> 检查连续天数条件 -> 检查成就解锁 -> 显示成就解锁动画如有

### 4.3 删除待办
悬停显示删除按钮 -> 点击删除 -> 如果是已完成项则总完成数减1 -> 从列表移除 -> 更新统计

## 6. 数据持久化

- todos: 待办事项列表 (JSON数组)
- stats: 统计数据 (JSON对象)
- 存储位置: 浏览器的 localStorage
